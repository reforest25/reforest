What is re:forest?

re:forest is an interactive and realistic forest growth simulator, aimed to raise awareness about deforestation. re:forest is also an educative and informative program.

Project Background

I chose to make a forest simulator for Codeavour 6.0 as global temperatures are rising, and that will cause problems for the near future. Deforestation is one of the main reasons behind rising global temperature. This problem needs to be resolved quickly.

Solution

The solution to solve rising global temperatures is to fight against it. To do so, more trees need to be planted. But not everyone knows and wants to do so. Which is why re:forest is created, a project that helps to raise awareness about this issue and could also be used to calculate useful outcomes for a forest or an individual tree.

Development Process

Development Process

1

Project Initiation

There were many problems contributing to rising global temperatures. Deforestation stood out as a key issue. I realized that reforestation is one of the easiest ways to fight against this. However, not everyone understands its importance. Which is why re:forest was created, a program to raise awareness of the impact trees can have in solving this problem.

2

Project Planning

Thinking outside of the box was crucial to create this project. Since the program needed to be user-friendly, the UI needed to be clean and simple, the features must be realistic but not too complex. Performance was also a crucial part of this program. These are the key ideas that shaped re:forest as a program. A way to plant trees is needed, which is why the program has tiles. Tree growth is another thing, one of the most important parts of this program, having many factors, and making those factors customizable was the plan.

3

Project Execution

With just around 20 days until the competition deadline, I began building the tile system, followed by data storage for each tile. I then developed the tree planting system, designed the UI, and created sprites. I implemented growth factors, including rain, seasons, soil quality, and moisture. A key feature was displaying simulation data like days, seasons, carbon reduction, and oxygen production. After completing the settings and adding a user guide, the program was ready.

4

Project Monitoring and Controlling

Once the project was complete, I asked for feedback from a few users. Based on their input, I refined the program and made necessary adjustments.

5

Project Closure

Before submitting, I created a banner and a video showcasing the project. With some extra time, I made final improvements to the program and submitted it to Codeavour.
